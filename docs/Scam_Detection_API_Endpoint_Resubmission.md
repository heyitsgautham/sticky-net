In this level, participants must resubmit their improved or corrected Scam Detection API endpoint along with the API key and GitHub repository. The evaluation system will invoke the endpoint again to verify fixes, improvements, and compliance with the required response format.

Resubmission Requirements

Required Submissions

    Updated Public API Endpoint URL – Must be live and accessible
    Valid API Key – Required for authentication
    Updated GitHub Repository URL – Must reflect latest fixes or improvements


Reference Document

    Agentic Honey-Pot for Scam Detection & Intelligence Extraction
    Refer this document: Scam Detection Reference Document

Note: Ensure your API strictly follows the request and response format defined in the reference document.

Important: Only ONE submission attempt is allowed for this resubmission level. Please read the documentation carefully and verify your endpoint before submitting.


Resubmission Guidelines

    Fix issues identified in the previous evaluation
    Ensure the endpoint is stable and reachable
    Response format must match the specification
    Repository should include clear setup and run instructions
    Do not change authentication mechanism


Evaluation Process

    System will re-run official test scenarios
    Authentication will be validated using the API key
    Response accuracy and structure will be verified
    Performance improvements will be considered


Outcome

    Final scores will be updated based on resubmission results
    Qualified submissions will move to the final ranking stage
